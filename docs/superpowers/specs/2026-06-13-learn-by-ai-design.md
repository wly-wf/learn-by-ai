# LearnByAI 设计文档

## 概述

**LearnByAI** 是一款基于 Tauri v2 的 Windows 桌面 AI 辅助学习工具。以"精读模式"为核心体验，用户可以在阅读文档时随时选中文字、截图，向 AI 追问内容。界面采用三栏布局：左侧目录大纲、中间阅读区、右侧 AI 对话面板。

## 目标用户

- 大学生/研究生：阅读论文、教材、文献
- 职场自学者：学习新领域知识（编程、考证、行业知识）
- 知识工作者：深度阅读和分析文档（研究员、分析师等）
- 兼顾通用性，任何想用 AI 辅助阅读的人都能使用

## 技术栈

| 层 | 技术 | 说明 |
|----|------|------|
| 桌面壳 | Tauri v2 | Rust 驱动，Windows 原生，体积 ~10MB |
| 前端框架 | React + TypeScript | 生态丰富，文档渲染库成熟 |
| UI 样式 | Tailwind CSS + shadcn/ui | 轻量可定制，无障碍支持 |
| 文档渲染 | pdf.js / mammoth.js / marked | 统一 Web 渲染 |
| AI 集成 | 通用 OpenAI 兼容接口 | 用户配置 Base URL + API Key |
| 本地存储 | IndexedDB（V1） | 对话历史、阅读进度 |
| 测试 | Vitest + Testing Library + Playwright | 单元/组件/E2E 全覆盖 |

## 架构

```
┌──────────────────────────────────────────────────────────┐
│  Tauri v2 桌面壳 (Rust)                                   │
│  ┌────────────────────────────────────────────────────┐  │
│  │  WebView 前端 (React + TypeScript)                   │  │
│  │  ┌─────────┐ ┌──────────────────┐ ┌─────────────┐  │  │
│  │  │ 大纲面板 │ │     阅读区        │ │  AI对话面板  │  │  │
│  │  │         │ │  (统一Web渲染)     │ │             │  │  │
│  │  │ 树形导航 │ │  pdf.js          │ │ 聊天界面     │  │  │
│  │  │ 双向同步 │ │  mammoth.js      │ │ 上下文卡片   │  │  │
│  │  │         │ │  marked          │ │ 截图粘贴     │  │  │
│  │  └─────────┘ └──────────────────┘ └─────────────┘  │  │
│  └────────────────────────────────────────────────────┘  │
│  Rust 后端：文件系统访问、本地存储、HTTP 请求代理          │
└──────────────────────────────────────────────────────────┘
                              │
                       云端 AI API
                       (用户自行配置)
```

- 所有文档格式统一转为 HTML 在 WebView 中渲染
- Rust 层负责文件读写、本地数据持久化、API 请求转发（避免前端暴露 API Key 到浏览器网络面板）
- 不内置 API Key，用户在设置中自行配置

## UI 布局

### 整体框架

```
┌─────────────────────────────────────────────────────────┐
│ LearnByAI    📂打开  📁导入文件夹    [📄论文][📝笔记]  ⚙️ │ ← 极简顶栏
├──────────┬──────────────────────────┬────────────────────┤
│ 📁 大纲   │  📖 阅读区                │  🤖 AI 对话         │
│ 树形目录  │  文档内容渲染              │  消息列表           │
│ 双向同步  │  选中文字 → 悬浮工具栏     │  上下文卡片         │
│          │                          │  输入框 + 截图按钮   │
├──────────┴──────────────────────────┴────────────────────┤
│  ←———————→ 可拖拽分隔条（1:3:2 默认比例）                  │
└─────────────────────────────────────────────────────────┘
```

### 三栏详解

**左侧大纲面板**
- 默认宽度约占窗口 1/6
- 树形结构展示文档目录
- 不同格式的大纲来源：
  - Markdown：自动解析 `#`/`##`/`###` 标题层级
  - PDF：优先提取内置书签（Bookmarks），无则 AI 自动生成
  - Word：解析 Heading 1/2/3 样式生成
  - TXT：无自动大纲，提供"AI 生成大纲"按钮按需触发
- 双向同步：点击标题 → 阅读区跳转；阅读区滚动 → 大纲自动高亮当前位置

**中间阅读区**
- 默认宽度约占窗口 1/2
- 统一 Web 渲染，所有格式转 HTML 展示
- 文字可选中（原生 Selection API）
- 选中后 300ms 延迟浮出工具栏：[💬追问] [📝笔记] [🔍解释]
- 选中后右键菜单包含：[AI提问] [AI翻译] [AI总结]
- 支持滚动，滚轮/触控板流畅翻页

**右侧 AI 面板**
- 默认宽度约占窗口 1/3
- 消息列表：用户消息（蓝色气泡）+ AI 回复（白色气泡）
- 上下文卡片：选中文字后以黄色卡片形式插入输入框上方，可添加多张、可删除
- 输入框：文本输入 + 📷 截图按钮 + 发送按钮
- 截图：支持 Ctrl+V 粘贴剪贴板图片 + 内置截图按钮
- 对话管理：每个文档绑定独立会话
- 未配置 API Key 时显示引导提示

### 顶栏

- 极简设计，高度约 36px
- 左侧：📂 打开文件 / 📁 导入文件夹
- 中间：文档标签页栏，可切换、关闭
- 右侧：⚙️ 设置齿轮

## 核心交互流

### 1. 选中文字 → AI 追问

```
阅读区选中文字
    │
    ├─ 300ms 延迟 → 悬浮工具栏 [💬追问] [📝笔记] [🔍解释]
    │                  │
    │              点击"追问"
    │                  │
    │          选中文字作为「上下文卡片」插入 AI 输入框上方
    │                  │
    │          用户输入问题 + 发送
    │                  │
    │          AI 回复（携带上下文）
    │
    └─ 或：右键 → [AI提问] [AI翻译] [AI总结] → 同上流程
```

### 2. 截图提问

```
系统截图工具（Win+Shift+S）截图
    │
Ctrl+V 粘贴 或 点击 📷 按钮
    │
图片预览卡片显示
    │
用户输入问题 + 发送
    │
图片 + 问题发往视觉模型 → AI 回复
```

### 3. 大纲联动

- 点击大纲标题 → 阅读区平滑滚动到对应章节
- 阅读区滚动 → 大纲自动高亮当前所在章节（蓝色背景）
- 支持折叠/展开子章节

### 4. 对话生命周期

- 打开文档 → 自动创建/恢复该文档的对话会话
- 切换标签页 → 自动切换对应对话
- 关闭标签 → 对话历史自动保存到 IndexedDB
- 再次打开同一文档 → 恢复对话历史 + 上次阅读位置

## 文件管理

- **导入方式**：拖拽文件到窗口、通过"打开"对话框选择、导入整个文件夹
- **支持格式**：txt、pdf、docx、md
- **标签页**：多文档以标签页形式管理，每个标签独立保存会话和阅读进度
- **大文件处理**：超过 100MB 显示进度条和提示，超大 PDF 逐页懒加载

## AI 配置

- 使用通用 OpenAI 兼容接口格式
- 用户配置项：
  - Base URL（如 `https://api.anthropic.com`、`https://api.openai.com`）
  - API Key
  - 模型名称（如 `claude-sonnet-4-6`、`gpt-4o`）
- 设置存储在本地，Rust 层代理转发请求（保护 API Key 安全）
- 支持设置多个提供商，快速切换

## 数据存储

**V1 使用 IndexedDB**，存储以下数据：

| 数据类型 | 说明 |
|----------|------|
| 文档元信息 | 文件路径、文件名、类型、最近打开时间 |
| 阅读进度 | 文档 ID + 滚动位置/页码 |
| 对话历史 | 文档 ID 绑定的消息列表 |
| 用户设置 | API 配置、界面偏好 |
| 大纲缓存 | AI 生成的大纲数据 |

后续可迁移到 SQLite（通过 Tauri 插件）以获得更好的查询性能。

## 异常处理

| 场景 | 处理方式 |
|------|----------|
| API Key 未配置 | AI 面板显示引导提示，输入框禁用 |
| API 调用失败 | 消息气泡标红 + 错误原因 + 重试按钮 |
| 大文件 (>100MB) | 加载进度条 + 提示 |
| 无法解析的文档 | "无法解析此格式"，提供"以纯文本打开"降级选项 |
| 网络断开 | AI 面板显示离线标识，输入框禁用，上下文保留 |
| 选中文字过长 (>5000字) | 自动截断，提示"已截取前 5000 字" |
| 截图过大 (>20MB) | 前端压缩后再发送 |
| 空文档 | 显示"文档为空" |

## 测试策略

| 层级 | 工具 | 覆盖范围 |
|------|------|----------|
| 单元测试 | Vitest | 文档解析、大纲生成、上下文截断、API 请求构造 |
| 组件测试 | Testing Library | 大纲面板交互、选中文字逻辑、AI 对话框 |
| E2E 测试 | Playwright | 完整流程：打开文档 → 选中 → 追问 → 回复 |
| 集成测试 | Tauri 测试 | 文件导入、本地存储读写 |

## 不做（V1 范围外）

- 多文档对比研究模式（V2）
- 本地模型部署（V2）
- 协作/分享功能
- 云同步
- 插件系统
- 移动端
